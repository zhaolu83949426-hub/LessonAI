# Lesson AI 需求文档（按当前实现对齐）

## 1. 文档目的

本文件用于同步当前仓库中已经落地的业务能力、页面结构、接口约束和数据模型，作为后续开发、联调和页面调整的基线。

- 以当前代码实现为准
- 不再保留未落地的假设性方案
- 若页面或接口后续继续调整，应同步更新本文件

## 2. 产品概览

### 2.1 产品定位

Lesson AI 是一个面向幼儿园老师的教案生成系统，核心目标是通过“模板 + 会话 + 大模型生成 + 结果编辑”的方式，缩短老师整理教案的时间。

### 2.2 当前版本核心能力

- 账号注册、登录、退出
- 模板新增、编辑、删除、列表查看
- 会话列表查看与继续编辑
- 新建会话并选择模板、模型
- 基于大模型生成教案内容
- 在会话详情页继续追问，持续更新当前教案
- 查看、编辑、保存教案结果
- 复制教案结果到剪贴板
- 首页删除会话
- 注册成功后自动登录
- 模板分类、标签与模板复用
- 我的页面与个人偏好设置
- 教案质量反馈

### 2.3 当前版本不包含的能力

- 后台管理端
- 模型配置管理页面
- 教案导出 Word / PDF
- 富文本编辑器
- 流式输出
- 模板共享
- 多角色权限体系

## 3. 使用角色与终端

### 3.1 使用角色

当前仅支持普通登录用户。

- 用户只能访问自己的模板、会话和教案记录
- 不区分管理员、运营、审核等角色

### 3.2 使用终端

- 前端为移动端优先页面
- 同时兼容 PC 浏览器访问
- 当前使用 Vue 3 + Vant 实现，页面交互偏 H5 风格

## 4. 信息架构与页面路由

### 4.1 当前页面清单

- `/login` 登录页
- `/register` 注册页
- `/` 首页，会话列表页
- `/templates` 模板列表页
- `/templates/:id/edit` 模板编辑页，`id=new` 时表示新建模板
- `/profile` 我的页面
- `/session/new` 新建会话页
- `/session/:id` 会话详情页
- `/session/:id/record` 教案结果编辑页

### 4.2 当前导航结构

- 首页和模板列表页使用底部导航布局
- 底部导航包含“会话”“模板”“我的”三个入口
- 新建会话页、会话详情页、教案结果页不显示底部导航
- “我的”页面承载资料与偏好设置

## 5. 核心业务流程

### 5.1 账号进入流程

1. 用户注册账号
2. 注册成功后前端直接保存 token 与用户信息
3. 自动进入首页会话列表
4. 后续访问继续通过本地 token 维持登录态

### 5.2 新建教案流程

1. 用户从首页进入新建会话页
2. 先选择模板
3. 再选择生成模型
4. 在底部输入框输入第一条需求，页面自动缓存草稿
5. 前端先创建会话，再调用生成接口
6. 后端保存本轮教案记录与聊天消息
7. 生成成功后前端跳转到会话详情页

### 5.3 会话追问流程

1. 用户进入会话详情页
2. 查看历史用户消息和 AI 回复
3. 在底部输入框继续追问
4. 前端展示阶段性生成状态，失败时保留输入内容
5. 后端基于当前会话最新教案内容重新生成完整教案
5. 生成完成后刷新会话详情与当前结果

### 5.4 教案编辑流程

1. 用户从新建页或会话详情页点击“查看教案”或“查看结果”入口
2. 进入教案结果编辑页
3. 页面默认加载当前会话下最新一条教案记录
4. 用户可切换查看原始生成稿和当前编辑稿
5. 用户可直接修改教案正文
6. 点击保存后更新该条记录的 `editedContent`
7. 用户可将当前教案内容复制到剪贴板
8. 用户可提交质量反馈

## 6. 功能需求

### 6.1 认证与登录态

#### 功能范围

- 用户注册
- 用户登录
- 获取当前登录用户信息
- 前端退出登录

#### 输入约束

- 注册账号 `account`：必填，长度不超过 32
- 注册昵称 `nickname`：必填，长度不超过 32
- 注册密码 `password`：必填，长度 6 到 32
- 登录账号 `account`：必填，长度不超过 32
- 登录密码 `password`：必填，长度不超过 32

#### 当前实现规则

- 账号唯一
- 密码使用加密哈希存储
- 登录成功后后端返回 token 与用户信息
- 注册成功后前端直接使用注册接口返回的 token 与用户信息完成自动登录
- 前端将 token 保存到 `localStorage`
- 需要登录的页面通过路由守卫校验 token
- 接口返回 401 时，前端清理 token 并跳转登录页
- 当前没有后端 `/logout` 接口，退出登录为前端本地清理行为

### 6.2 首页与会话列表

#### 功能范围

- 查看当前用户的会话列表
- 从首页进入新建会话页
- 从首页进入会话详情页
- 按前端内置条件筛选会话
- 删除会话
- 关键词搜索与更新时间排序

#### 当前页面行为

- 首页显示欢迎语、快捷入口卡片、最近会话区块
- 会话卡片点击后进入对应会话详情页
- 页面提供四个筛选项：全部、已生成、草稿、今日更新
- 会话状态依据 `currentResultId` 是否存在进行判断
- 页面支持按标题、模板名、模型名搜索
- 页面支持按最近更新时间正序或倒序切换
- 会话卡片直接展示模板名、模型名、最近提问摘要，并提供删除入口

#### 列表规则

- 后端按会话 `updatedAt` 倒序返回
- 已生成会话：`currentResultId` 不为空
- 草稿会话：`currentResultId` 为空
- 今日更新：前端基于 `updatedAt` 的日期判断
- 删除前端先二次确认，确认后调用删除接口并本地刷新列表

#### 当前限制

- 当前搜索和筛选仍是前端内存过滤，不是服务端分页检索

### 6.3 模板管理

#### 功能范围

- 模板列表
- 新建模板
- 编辑模板
- 删除模板
- 模板分类与标签
- 模板复用

#### 字段约束

- 模板名称 `name`：必填，长度不超过 50
- 模板内容 `content`：必填，长度不超过 10000
- 模板分类 `category`：选填，长度不超过 32
- 模板标签 `tags`：选填，长度不超过 255

#### 当前实现规则

- 模板仅当前登录用户可见
- 模板列表按 `updatedAt` 倒序返回
- 模板编辑页同时承担“新建”和“编辑”两种场景
- 删除模板需要二次确认
- 模板支持填写分类和标签
- 模板列表支持基于现有模板快速复制出“复用模板”
- 当前没有单独的模板只读详情页

### 6.4 大模型选择

#### 功能范围

- 获取可用模型列表
- 获取默认模型
- 在新建会话前选择模型

#### 当前实现规则

- 前端仅展示可用模型，不提供模型管理能力
- 后端只返回 `enabled = true` 的模型配置
- 默认模型来自数据库中的 `is_default = true` 配置
- 如果数据库中没有默认模型，启动阶段会按配置补充默认模型
- 当前创建会话时确定模型，后续在该会话中继续追问时沿用会话已有模型

#### 当前返回字段

- `id`
- `name`
- `modelCode`
- `isDefault`

### 6.5 会话管理

#### 功能范围

- 新建会话
- 查看会话列表
- 查看会话详情
- 查看会话消息
- 删除会话

#### 会话创建规则

- 创建会话时必须传入模板 ID
- 创建会话时必须传入模型配置 ID
- 创建会话时必须传入标题
- 当前前端默认使用首条输入的前 15 个字符作为会话标题
- 新建会话页会本地缓存当前输入、模板和模型选择

#### 会话详情展示内容

- 会话标题
- 模板名称
- 模型名称
- 聊天消息列表
- 当前结果入口

#### 消息规则

- 消息按创建时间正序返回
- 每条消息包含角色、内容、关联教案记录 ID、创建时间
- 当前角色值使用 `USER` 与 `ASSISTANT`
- 前端根据角色渲染左右两侧气泡

#### 当前限制

- 当前前端没有“修改会话模板”或“修改会话模型”的入口
- 当前没有会话级高级筛选接口

### 6.6 教案生成

#### 输入参数

- `sessionId`：必填
- `topic`：必填，长度不超过 100
- `userMessage`：必填，长度不超过 5000

#### 当前生成策略

首次生成时，后端使用以下信息拼装 prompt：

- 模板内容
- 固定教案输出结构要求
- 当前主题
- 当前用户输入

追问生成时，后端不直接回放完整聊天历史，而是基于以下信息重新生成完整教案：

- 当前最新教案内容
- 固定教案输出结构要求
- 老师本轮最新补充要求

#### 固定输出结构

当前后端固定要求模型输出以下六个部分：

- 活动名称
- 活动目标
- 活动准备
- 活动过程
- 延伸活动
- 注意事项

#### 落库规则

每次生成后会保存：

- 教案记录 `lesson_records`
- 用户消息 `session_messages`
- AI 回复 `session_messages`
- 会话当前结果 ID `chat_sessions.current_result_id`

#### 当前记录内容

- 会话 ID
- 模板 ID
- 模型配置 ID
- 模型名称快照
- 主题
- 输入快照
- 最终 prompt
- 原始生成结果
- 可编辑结果内容

#### 生成模式

- `INITIAL`：当前会话尚无历史教案结果
- `FOLLOW_UP`：当前会话已有历史教案结果

#### 当前限制

- 当前为同步生成
- 当前没有流式输出
- 当前没有中断生成、重试、版本回滚等机制
- 当前生成进度提示仍为前端阶段性文案，不包含服务端真实进度百分比

### 6.7 教案结果编辑

#### 功能范围

- 查看当前会话最新教案结果
- 编辑教案正文
- 保存教案正文
- 复制教案正文
- 查看模板名称
- 查看原始生成稿
- 提交质量反馈

#### 当前实现规则

- 页面通过会话 ID 加载该会话下全部教案记录
- 当前前端默认取最后一条记录作为编辑对象
- 编辑框默认优先显示 `editedContent`，为空时回退为 `resultContent`
- 保存时仅更新 `editedContent`
- 页面提供复制到剪贴板能力
- 页面支持切换“当前编辑稿 / 原始生成稿”
- 页面支持提交评分、质量等级和补充反馈

#### 当前页面展示情况

- 能稳定展示模板名称、模型名称、更新时间
- 页面支持返回会话详情继续追问
- 页面支持质量反馈录入

## 8. 接口清单

### 8.1 通用返回结构

所有接口统一返回：

```json
{
  "success": true,
  "data": {},
  "message": "OK"
}
```

业务失败时：

- `success = false`
- `message` 返回错误原因

### 8.2 认证接口

- `POST /api/auth/register` 注册
- `POST /api/auth/login` 登录
- `GET /api/auth/me` 获取当前用户信息
- `PUT /api/auth/me` 更新当前用户资料
- `PUT /api/auth/password` 修改当前用户密码

说明：当前仅 `POST /api/auth/register` 和 `POST /api/auth/login` 为匿名可访问接口，其余接口均要求认证。

### 8.3 模板接口

- `GET /api/templates` 模板列表
- `POST /api/templates` 新建模板
- `GET /api/templates/{id}` 模板详情
- `PUT /api/templates/{id}` 编辑模板
- `DELETE /api/templates/{id}` 删除模板
- `POST /api/templates/{id}/duplicate` 复制模板

### 8.4 模型接口

- `GET /api/llm-configs/options` 获取可选模型列表
- `GET /api/llm-configs/default` 获取默认模型

### 8.5 会话接口

- `GET /api/sessions` 会话列表
- `POST /api/sessions` 新建会话
- `GET /api/sessions/{id}` 会话详情
- `GET /api/sessions/{id}/messages` 会话消息列表
- `DELETE /api/sessions/{id}` 删除会话

### 8.6 教案记录接口

- `POST /api/lesson-records/generate` 生成教案
- `GET /api/lesson-records/session/{sessionId}` 获取会话下全部教案记录
- `PUT /api/lesson-records/{id}` 更新教案编辑结果
- `PUT /api/lesson-records/{id}/feedback` 更新教案质量反馈

## 11. 安全与数据隔离

### 11.1 安全要求

- 登录接口使用 JWT 返回登录态
- 认证接口之外的业务接口统一要求登录
- 密码不得明文存储
- 大模型 API Key 仅保存在服务端配置和数据库中，不返回前端

### 11.2 数据隔离要求

- 模板按用户 ID 隔离
- 会话按用户 ID 隔离
- 教案记录通过所属会话与用户关系校验访问权限
- 非本人数据不可查询、不可修改、不可删除

## 12. 当前实现差异与注意事项

### 12.1 已对齐到代码的关键点

- 当前会话追问并不是拼接完整聊天历史，而是基于“最新教案内容 + 最新补充要求”重新生成完整结果
- 当前注册成功后会自动登录并进入首页
- 当前退出登录为前端本地行为，没有后端退出接口
- 当前首页已支持会话删除、搜索、排序和摘要展示
