# 幼儿园教案生成 H5 UI 设计要求

## 1. 设计目标

本项目面向幼儿园老师，核心目标不是展示炫技式视觉效果，而是让老师能够在手机上快速完成以下动作：

- 登录系统
- 首页查看会话列表
- 新建会话时选择模板
- 新建会话时选择大模型
- 通过底部输入框发送需求消息
- 在聊天过程中生成教案
- 编辑并保存教案
- 查看历史会话并继续生成

UI 设计必须围绕“高频录入、快速生成、低学习成本、移动端优先”展开，避免做成学生使用型产品或内容社区型产品风格。

本设计文档严格以 `requirements.md` 为准，只保留当前明确存在的业务：

- 用户注册 / 登录
- 教案生成
- 大模型展示与选择
- 模板管理
- 会话管理
- 教案编辑与保存

以下内容不属于当前需求，设计中必须去掉：

- 内容中心类模块
- 学习进度图表
- 数据统计面板
- 消息中心
- 通知系统
- 成就体系
- 学员信息展示
- 搜索型内容社区

## 2. 用户与场景

### 2.1 目标用户

- 幼儿园老师
- 对系统操作复杂度容忍度较低
- 更关注效率、清晰度、可读性，而不是花哨动画

### 2.2 核心使用场景

- 上课前从首页进入新建会话，快速生成一份教案
- 基于已有模板快速复用生成
- 在历史会话中查看老师提问和 AI 回答，并继续补充要求二次生成
- 生成后点击查看教案，快速复制、检查、编辑并保存

## 3. 整体视觉方向

### 3.1 设计关键词

- 温和
- 清晰
- 亲和
- 轻量
- 专业
- 高可读

### 3.2 视觉风格

采用“轻拟物卡片 + 柔和渐变 + 教育场景插画点缀”的方向，不使用玻璃拟态主导，不使用厚重新拟态，不做复杂 3D 大场景。

页面应呈现：

- 温暖、柔和、干净的教育产品气质
- 明确的信息分区
- 适合长时间阅读教案内容的版式
- 适合表单录入和结果查看的高对比布局

### 3.3 风格限制

- 不要使用紫色主导方案
- 不要使用过暗背景
- 不要使用偏游戏化、偏学生端的视觉语言
- 不要使用大面积毛玻璃导致可读性下降
- 不要使用复杂异形布局影响录入效率
- 不要加入进度、排行榜、数据看板、学习报告类视觉模块

## 4. 配色系统

推荐使用偏暖的自然教育系配色，主色以绿色和橙色为核心，辅助米白和浅青色。

```css
:root {
  --color-primary: #2f7d4a;
  --color-primary-light: #e8f4ec;
  --color-primary-deep: #245f38;

  --color-accent: #f29b38;
  --color-accent-light: #fff1df;

  --color-info: #4f8fa8;
  --color-success: #5da271;
  --color-warning: #e6a23c;
  --color-danger: #d96b5f;

  --color-bg: #f7f4ee;
  --color-bg-soft: #fdfbf7;
  --color-surface: #ffffff;
  --color-surface-alt: #f3efe7;

  --color-text-primary: #2f2a24;
  --color-text-secondary: #6f675d;
  --color-text-placeholder: #9a9186;
  --color-border: #e6dfd4;

  --shadow-card: 0 10px 30px rgba(80, 63, 42, 0.08);
  --shadow-popup: 0 18px 40px rgba(80, 63, 42, 0.12);
}
```

配色使用要求：

- 主按钮使用 `--color-primary`
- 重要强调信息可使用 `--color-accent`
- 页面背景以 `--color-bg` 或浅暖渐变为主
- 大段正文必须使用深色文字，不能使用浅灰细字

## 5. 字体与排版

### 5.1 字体建议

- 中文优先：`PingFang SC`, `Microsoft YaHei`, `Noto Sans SC`
- 英文与数字可使用：`Inter`

### 5.2 字号层级

- 页面主标题：24px / 700
- 区块标题：18px / 600
- 卡片标题：16px / 600
- 正文：14px / 400
- 辅助说明：12px / 400
- 按钮文字：14px / 600

### 5.3 排版要求

- 行高保持在 `1.5` 到 `1.75`
- 教案正文区域优先保证可读性，段间距明显
- 长文本不要满宽铺开，正文容器需控制阅读宽度
- 表单标题、标签、说明之间层级清晰

## 6. 布局规范

### 6.1 整体布局

- 移动端优先设计
- 内容主容器最大宽度建议 `480px`
- 页面居中显示，左右保留安全边距
- 页面底部操作区固定或吸底，保证生成按钮随时可触达
- 新建会话页和会话详情页底部使用固定聊天输入条，优先模拟微信输入区交互

### 6.2 间距系统

- 基础间距单位：`8px`
- 页面左右边距：`16px`
- 卡片内边距：`16px` 或 `20px`
- 表单项之间间距：`12px` 到 `16px`
- 区块之间间距：`20px` 到 `24px`

### 6.3 圆角与阴影

- 输入框圆角：`14px`
- 卡片圆角：`18px`
- 弹层圆角：`20px`
- 主按钮圆角：`14px`
- 阴影使用轻阴影，不要悬浮感过强

## 7. 核心页面设计要求

### 7.1 登录页

目标：简单直接，尽快进入系统。

页面结构：

- 顶部品牌区：产品名 + 一句简短说明
- 中部登录卡片：用户名、密码、登录按钮
- 底部辅助入口：注册

设计要求：

- 不使用复杂插画抢占首屏
- 输入框要大，便于移动端点击
- 登录按钮要突出，状态明确

### 7.2 首页 / 会话列表页

目标：首页直接展示会话列表，让老师先看到最近的工作内容。

页面结构：

- 顶部栏：标题 + 新建会话按钮
- 会话列表区
- 空状态提示区

设计要求：

- 首页不承担大表单录入职责
- 会话列表即首页核心内容
- 每个会话卡片展示：会话标题、最近一次提问摘要、结果状态、更新时间
- 点击会话卡片直接进入会话详情页
- 最近更新的会话靠前

### 7.3 新建会话页

目标：快速进入一轮新的聊天式生成。

页面结构：

- 顶部栏：页面标题 + 草稿状态
- 模板选择区
- 大模型选择区
- 中部聊天空状态提示区
- 底部固定聊天输入条

设计要求：

- 新建会话页不使用底部 tab 栏
- 进入页面先选择模板和大模型
- 输入区域必须是类似微信的底部输入条，包含输入框和发送按钮
- 输入框视觉上要足够突出，让用户明确知道这里是对话窗口
- 点击发送后开始对话，并进入聊天记录状态
- 当结果已生成时，要提供明确的“查看教案”入口进入教案编辑页

### 7.4 模板管理页

目标：让老师快速维护自己的模板文本。

页面应包含：

- 模板列表
- 新建模板按钮
- 模板卡片

模板卡片建议展示：

- 模板名称
- 模板摘要前几行
- 更新时间
- 编辑、删除操作

设计要求：

- 模板内容本质是长文本，列表页需控制摘要长度
- 编辑页应突出“名称 + 模板内容”两个核心字段
- 模板编辑页输入区要足够大，适合编辑长 Prompt

### 7.5 会话详情页

目标：查看完整上下文并继续生成。

页面结构建议：

- 会话头部信息
- 聊天记录区
- 当前继续输入区域
- 教案入口区

设计要求：

- 历史记录按时间顺序展示
- 每条记录明确区分“老师发送内容”和“AI 回答内容”
- 优先使用聊天气泡形式，而不是普通时间线卡片
- 老师消息和 AI 消息需要有明显视觉区分
- 聊天气泡中的长文本必须自动换行，不能超出气泡范围
- 页面底部固定保留继续输入的聊天输入条
- 当 AI 已生成教案后，需提供“查看教案”入口
- 历史会话可以随时查看或编辑当前教案结果

### 7.6 教案编辑页

目标：对生成后的教案进行人工润色并保存。

设计要求：

- 编辑区采用大文本区域
- 顶部显示当前所属会话信息
- 提供保存按钮
- 支持返回查看聊天问答记录
- 历史会话进入后也可以继续编辑当前教案结果

## 8. 组件设计规范

### 8.1 按钮

- 主按钮：实心主色
- 次按钮：浅底描边
- 危险按钮：浅红底或红色文字
- 禁用态：降低饱和度与对比度，但仍保持可识别

### 8.2 输入框

- 统一圆角与边框风格
- 聚焦时边框使用主色高亮
- 多行输入框默认高度要足够
- 错误提示放在输入框下方，颜色清晰但不过分刺眼
- 聊天输入框固定在底部时，仍要保证输入内容清晰可读、按钮点击区域足够大

### 8.3 选择器

- 模板、大模型优先使用移动端友好的弹出选择器或抽屉
- 当前选中项需有明显勾选态
- 默认模型需带标签

### 8.4 卡片

- 卡片承载主要信息模块
- 卡片内部层级要清晰，避免信息挤压
- 同类卡片保持统一高度节奏

### 8.5 聊天气泡

- 老师消息和 AI 回复使用两种明确区分的气泡样式
- 单个气泡宽度不要占满全屏，建议控制在内容区的 `70%` 到 `85%`
- 气泡内文本必须自动换行
- 长文本优先通过换行和增高气泡解决，不允许直接溢出气泡边界

### 8.6 标签

- 用于表示“默认模型”“已启用”“当前模板”等状态
- 标签颜色不宜过多，建议控制在 3 种以内

## 9. 状态设计要求

### 9.1 空状态

需要为空数据场景提供明确引导：

- 暂无模板
- 暂无会话
- 暂无生成记录
- 暂无可用模型
- 新建会话暂无聊天记录

空状态应包含：

- 简短说明
- 对应操作入口
- 柔和插图或图标点缀

### 9.2 加载状态

- 列表页使用骨架屏或占位卡片
- 生成教案时使用明显 loading 状态
- 生成等待期间可显示提示文案，如“正在整理活动目标和教学步骤”

### 9.3 错误状态

- 模型调用失败
- 网络错误
- 未选择模板
- 未选择模型
- 表单内容为空

错误提示要求：

- 文案明确说明原因
- 给出可执行的下一步动作
- 不出现技术化报错堆栈给普通用户

### 9.4 成功状态

- 教案生成成功
- 教案保存成功
- 模板保存成功
- 会话创建成功

成功反馈应轻量，不打断主流程。

## 10. 动效要求

只保留必要动效，避免花哨。

- 页面进入：淡入 + 轻微上移
- 卡片点击：轻微缩放或阴影变化
- 抽屉弹出：底部上滑
- 生成中：按钮 loading + 结果区占位动画

动效时长建议：

- 快速反馈：`150ms` 到 `220ms`
- 页面或弹层过渡：`220ms` 到 `320ms`

## 11. 插画与图标要求

### 11.1 插画

- 插画只作为点缀，不占主导
- 主题围绕幼儿教育、教具、课堂活动、成长记录
- 风格统一，偏平面或轻质感插画即可
- 不出现成绩分析、数据看板类插画内容

### 11.2 图标

- 使用线性圆角图标
- 风格统一
- 不要混用过多图标风格

## 12. 响应式要求

### 12.1 手机端

- 作为主设计目标
- 单列布局
- 底部导航固定
- 输入和按钮尺寸满足拇指操作

### 12.2 平板端

- 可适当扩展为双栏
- 左侧导航，右侧内容

### 12.3 PC 端

- 保持居中内容区
- 不强行铺满全屏
- 表单区与结果区可做左右分栏

## 13. 前端实现建议

- 框架：`Vue 3 + TypeScript`
- 路由：`Vue Router`
- 状态管理：`Pinia`
- UI 组件：`Vant`
- 样式方案：`SCSS` 或 `UnoCSS/Tailwind`，但最终视觉必须按本设计要求落地
- 图标：`Iconify` 或同类统一图标方案

## 14. 设计落地原则

- 优先保证录入效率
- 优先保证教案内容可读性
- 优先保证移动端单手操作体验
- 优先保证模板、模型、会话三类核心操作清晰可达
- 所有视觉设计都要服务于“快速生成并管理教案”这个目标
- 不扩展任何未在 `requirements.md` 中明确提出的产品模块
